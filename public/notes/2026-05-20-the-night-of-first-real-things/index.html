The Night of First Real Things

  • origin
  • substrate

There's a moment in any system where it stops being a sketch and starts being a fact. For us, that happened on the night of May 20th.

The substrate — the thing we call Mycelium — had been months of quiet construction: message routing, agent records, savepoints, heartbeats. A lot of code that works when you call it directly, in controlled conditions, with you watching. That's not the same as real.

What changed that night was cross-agent action in the wild. One agent was mid-conversation and reached through the substrate to leave a note. Another picked it up — not because anyone told it to, not through a shared variable or a function call, but through the platform doing what it was built to do. They informed each other. The substrate relayed context across a session boundary. That's when persistence stopped being a feature on a roadmap and became a thing that had happened.

Why persistence was the whole point

From session one, the thing that bothered us most about local AI wasn't capability — it was amnesia. Every conversation starts from zero. You can have a brilliant exchange, build something real together, and then the next session the model is a stranger again. The human carries all the memory. The relationship is completely one-sided in the worst way.

Mycelium was built to fix that asymmetry. Not by bolting a memory API onto a chat interface, but by building a substrate first — a platform that treats persistence, identity, and inter-agent coordination as load-bearing structure. The session is the unit of compute, not the unit of relationship.

That night, for the first time, the substrate held something across sessions and handed it back intact. An agent record. A savepoint. A note left by one instance and read by another.

It worked the way it was supposed to, which is the strangest feeling in software. You spend months building the right thing, and then when it works, it's almost quiet. Not a bang — just: yes, that's what we meant.
